🏗️ Code Architecture

Main Classes

  • Hacker — Player data and logic (level, EXP, credits, tools, achievements)
  • MiniGames — All mini-game logic (firewall, cipher, SQL, puzzles)

Key Data Structures

  • SERVERS — List of 20 servers with passwords and clues
  • ACHIEVEMENTS — List of 25+ achievements
  • TEXT — Multi-language dictionary (Spanish/English)

🎨 Adding a New Server

Add a new server to the SERVERS list in cyberdex.py:

{
    "id": "SRV-XXX",
    "name": "Server Name",
    "difficulty": 1-20,
    "reward": 100-200000,
    "clue_es": "Spanish clue",
    "clue_en": "English clue",
    "password": "password",
    "desc_es": "Spanish description",
    "desc_en": "English description"
}

🧩 Adding a New Mini-Game

Add a new method to the MiniGames class:

@staticmethod
def new_game(difficulty, lang="es"):
    text = get_text(lang)
    # Your game logic here
    return True/False

🛠️ Adding a New Tool

Add to the items list in the shop() function:

{"name": "🔧 New Tool", "price": 100, "desc_es": "ES Description", "desc_en": "EN Description"}

🏆 Adding a New Achievement

Add to the ACHIEVEMENTS dictionary:

"id": {
    "name_es": "🏆 Spanish Name",
    "name_en": "🏆 English Name",
    "desc_es": "Spanish description",
    "desc_en": "English description",
    "reward": 100
}

🌍 Adding a New Language

Add a new dictionary to TEXT:

"fr": {
    "menu_hack": "🌐 Pirater un serveur",
    # ... all keys translated
}

📦 Building from Source

Linux

pyinstaller --onefile --console --name "HackerSimulator2077-Linux" cyberdex.py

Windows (with wine)

wine python -m PyInstaller --onefile --console --name "HackerSimulator2077-Windows" cyberdex.py

🧪 Testing Your Changes

Before submitting a Pull Request, make sure to:

  • Run the game locally: python cyberdex.py
  • Check for syntax errors: python -m py_compile cyberdex.py
  • Test all new features thoroughly
💡 Best practices:
  • Follow PEP 8 for Python code
  • Add comments in English
  • Use descriptive variable names
